Mobile Options Design:

  • ✅ Textarea reduced from 6 rows to 4 (makes room for options)
  • ✅ 2-column grid for expiration chips (3 columns doesn't fit 375px)
  • ✅ Button chips: 48px height (thumb-friendly tap targets)
  • ✅ Selected chip (7 days): Blue background + checkmark icon
  • ✅ Passphrase hint simplified: "💡 Share separately via SMS or call"
  • ✅ "Hide options" link collapses panel back to initial state
  • ✅ Sticky button remains at bottom, above keyboard

Touch vs Desktop:

  • ✅ Hover → Active: Buttons use :active instead of :hover
  • ✅ Dropdowns → Chips: No tiny dropdown arrows to tap
  • ✅ 2 columns: Maximizes tap area while fitting screen
  • ✅ Larger gaps: 8px gap between buttons (prevents mis-taps)
  • ✅ Text size: text-base (16px) prevents iOS auto-zoom

Morgan's Use Case (Freelancer, Mobile):

This design solves Morgan's friction points from Phase 2:

  • ✅ Button chips instead of dropdown (no mis-taps)
  • ✅ Larger touch targets (48px minimum)
  • ✅ Sticky button visible when keyboard opens
  • ✅ Passphrase guidance inline (share separately)
  • ✅ Current flow: 6 clicks, ~45 seconds (vs 8 clicks, 60s before)
