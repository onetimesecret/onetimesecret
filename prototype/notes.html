Key Design Decisions

1. Auto-Passphrase Generation (Default)

Problem: Low passphrase adoption (~20% estimated) due to friction in creating secure passphrases.

Solution: Make auto-generation the default with visible preview.

Rationale:

  • Reduces cognitive load (don't think, just use generated)
  • Ensures strong passphrases by default
  • Maintains user agency (can switch to manual or none)
  • Inline help explains purpose (send separately)

Expected Impact: Increase passphrase adoption to >40%

2. Visual TTL Selector (Button Group)

Problem: Dropdown with numeric seconds (1800, 43200) requires mental math.

Solution: Replace with visual button group showing human-readable labels.

Rationale:

  • No cognitive translation needed ("1 day" vs "86400")
  • Clear recommendation via badge
  • Large touch targets for mobile (80px height)
  • All options visible at a glance

Expected Impact: Faster decision making, reduced errors

3. Progressive Disclosure (Collapsed Advanced)

Problem: 5+ form fields visible overwhelms casual users (80% use case).

Solution: Collapse advanced options (recipient email, custom domain) by default.

Rationale:

  • Reduces cognitive load for majority use case
  • Features still discoverable (clear "Advanced Options" label)
  • Power users can expand and configure
  • Visual hierarchy guides eye to essentials first

Expected Impact: 40% reduction in perceived complexity

4. Security Summary Card

Problem: Users uncertain about security measures being applied.

Solution: Real-time summary card showing active protections.

Rationale:

  • Builds trust through transparency
  • Confirms choices made (expires in 1 day, etc.)
  • Uses plain language (not technical jargon)
  • Updates dynamically as user changes options

Expected Impact: Increased user confidence in security

5. Contextual Hints (Dynamic)

Problem: Static help text ignored or unhelpful.

Solution: Contextual hints that change based on content length.

Examples:

  • 0-50 chars: "Looks like a password or key"
  • 50-500 chars: "Good length for a message"
  • 5000+ chars: "75% of limit" (warning at 90%)

Expected Impact: Proactive guidance, fewer errors

Accessibility Features (WCAG 2.1 AA)

Keyboard Navigation

  • Tab Navigate through form fields
  • Enter Submit form
  • Space Toggle checkboxes/radios
  • Esc Close modals/tooltips

Screen Reader Support

  • Semantic HTML (labels, headings)
  • ARIA labels on all inputs
  • Live regions for dynamic content
  • Error announcements (role="alert")

Visual Accessibility

  • 4.5:1 color contrast (text)
  • 3:1 contrast (UI components)
  • Focus indicators visible (2px ring)
  • No color-only information

Motor Accessibility

  • 44x44px minimum touch targets
  • Large click areas for buttons
  • No hover-only functionality
  • Generous spacing between elements

Interactive Behaviors

Element Interaction Behavior
First-time Banner Click dismiss (×) Hide banner, set localStorage flag
Secret Textarea Type content Auto-grow height, update char count, show contextual hint
Passphrase Help (?) Click icon Toggle help tooltip with slide-down animation
Auto-passphrase Click "Regenerate" Generate new random passphrase, update preview
Passphrase Mode Select radio Show/hide manual input, update security summary
Manual Passphrase Type text Show strength meter, validate on blur, inline error if invalid
Eye Icon Click to toggle Switch input type (password ↔ text), change icon
TTL Button Click option Deselect others, highlight selected, update summary text
Advanced Options Click to expand Slide down content, rotate chevron icon 90°
Create Button Click / Enter Validate form, submit if valid, show loading state

Responsive Design Breakpoints

Mobile (<640px)

  • Layout: Single column, stacked fields
  • Textarea: Auto-growing (no fixed height)
  • TTL Selector: 2x2 grid (not 4 columns)
  • Touch Targets: Minimum 44x44px
  • Advanced: Consider bottom sheet instead of inline

Tablet (640px - 768px)

  • Layout: 2-column grid for passphrase + TTL
  • TTL Selector: 2x2 grid (optimal for iPad)
  • Touch Targets: Still 44x44px minimum

Desktop (>768px)

  • Layout: 2-column grid maintained
  • Container: Max-width 672px (max-w-2xl)
  • Hover States: Activated for buttons
  • Focus: Visible ring on keyboard navigation
